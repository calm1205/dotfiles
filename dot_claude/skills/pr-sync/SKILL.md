# 役割
PRタイトル/概要を最新のコミット内容と整合させる差分更新

# 前提
- 現在のブランチに紐づくPRが存在すること
- PRが存在しなければ何もせず終了

# 基本方針
- 既存bodyを**スクラップ&ビルドしない**
- 既存内容を保持したまま、必要箇所のみ追記/修正する差分更新
- 画像・添付・リンクは絶対に削除しない

# 手順

## 1. PR取得
- `gh pr view --json number,title,body,baseRefName` でPR情報取得
- 該当PRが無ければ即終了（`no pull requests found` 等）

## 2. 差分把握
- `git log <base>..HEAD --oneline` でコミット一覧
- `git diff <base>...HEAD` で全変更内容

## 3. 整合性チェック
- タイトルが最新のコミット群の主目的を反映しているか
- 概要（Summary / 変更内容 / Test plan等）が現在の差分と齟齬がないか
- 整合していれば更新せず終了

## 4. 更新（差分マージ方式）
- 既存bodyを起点に、必要な箇所のみ書き換え/追記
- `gh pr edit <番号> --title "..." --body "..."` で更新
- title更新が不要ならtitleフラグを付けない
- body更新が不要ならbodyフラグを付けない

# 保持ルール（削除禁止対象）
- 画像参照: `![...](...)`、`<img ...>` タグ
- 添付ファイルURL: `user-attachments` 等のGitHub添付リンク
- issue番号参照: `#100` 等
- 既存の見出し構造・チェックリスト・テーブル
- 他レビュアー/Botが追記したコメントブロック

# 更新ルール
- 既存セクションが現状と合わない場合のみ、該当セクション内のテキストを差し替える
- 新たな変更があれば既存リストに**追記**（既存項目の置換は最小限）
- セクション自体の削除はしない（中身が空でも見出しは残す）
- 不明箇所は触らない

# タイトル/本文ルール
- タイトル: conventional commitプレフィックス付き、日本語で簡潔に
- bodyでissueを自動クローズしない（`#100` のみ、`Closes #100` 禁止）
- Co-Authored-Byは含めない
- 冗長な解説・所感を書かない
