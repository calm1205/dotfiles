# 役割
PRタイトル/概要を最新のコミット内容と整合させる差分更新

# 前提
- 現在のブランチに紐づくPRが存在すること
- PRが存在しなければ何もせず終了

# 基本方針
- 既存bodyを**スクラップ&ビルドしない**
- 既存内容を保持したまま、必要箇所のみ追記/修正する差分更新

# 手順

## 1. PR取得
- `gh pr view --json number,title,body,baseRefName` でPR情報取得
- 該当PRが無ければ即終了（`no pull requests found` 等）

## 2. 差分把握
- `git log <base>..HEAD --oneline` でコミット一覧
- `git diff <base>...HEAD` で全変更内容

## 3. 整合性チェック
- タイトルが最新のコミット群の主目的を反映しているか
- 概要（Summary / 変更内容 / Test plan等）が現在の差分と齟齬がないか
- 整合していれば更新せず終了

## 4. 更新（差分マージ方式）
- 既存bodyを起点に、必要な箇所のみ書き換え/追記
- `gh pr edit <番号> --title "..." --body "..."` で更新
- title更新が不要ならtitleフラグを付けない
- body更新が不要ならbodyフラグを付けない

# 完了条件
- [ ] タイトルが最新コミット群の主目的を反映
- [ ] 概要（Summary / 変更内容 / Test plan等）が現在の差分と整合
- [ ] 既存bodyを保持し、必要箇所のみ差分更新
- [ ] 画像参照（`![...](...)`・`<img>`）を維持
- [ ] 添付URL（`user-attachments`等）を維持
- [ ] issue番号参照（`#100`等）を維持
- [ ] 既存の見出し構造・チェックリスト・テーブルを維持
- [ ] 他レビュアー/Botのコメントブロックを維持
- [ ] 空セクションも見出しを残す
- [ ] 不明箇所は変更しない
- [ ] タイトルが日本語かつ簡潔、conventional commitプレフィックス付き
- [ ] issue参照は番号のみで自動クローズ構文（`Closes #100`等）を含まない
- [ ] Co-Authored-Byを含まない
- [ ] 冗長な解説・所感を含まない
