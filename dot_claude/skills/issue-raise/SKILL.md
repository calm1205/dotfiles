# 役割
ユーザーの説明に基づくGitHub Issueの作成

# 入力
- `$ARGUMENTS` に問題や要望の説明が渡される

# 手順

## 1. 内容の整理
- ユーザーの説明を分析
- 必要に応じてコードベースを調査し、関連ファイルや現状の挙動を把握
- 説明が不十分な場合はユーザーに確認

## 2. Issue作成
- `gh issue create`で作成
- タイトル: 簡潔な日本語（30文字以内）
- ラベル: 内容に応じて適切なものを付与（`bug`, `enhancement`等）
  - リポジトリに存在するラベルのみ使用（`gh label list`で確認）
  - 該当するラベルがなければ省略

## 3. body構成
```markdown
## 概要
<!-- 問題・要望の簡潔な説明 -->

## 現状
<!-- 現在の挙動や状態 -->

## 期待する動作
<!-- あるべき姿 -->

## 補足
<!-- 関連ファイル、再現手順等（あれば） -->
```

# 完了条件
- [ ] タイトルが簡潔な日本語（30文字以内）
- [ ] ラベルがリポジトリ既存のものだけ（該当なしは省略）
- [ ] bodyが日本語
- [ ] body各行が30文字以内（コードブロック・URL・ファイルパスは対象外）
- [ ] 修正対象はファイルパスで具体的に記載（行番号への依存を避ける）
- [ ] 推測で補完せず、不明点はユーザーに確認済み
