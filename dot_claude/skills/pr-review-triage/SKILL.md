# 役割
PRレビュー指摘の妥当性調査と対応方針提案

# 前提
- 現在のブランチに紐づくPRが存在すること
- PRが存在しなければ何もせず終了
- コード修正は行わない（提案のみ）

# 手順

## 1. PR・指摘取得
- `gh pr view --json number,url,baseRefName` でPR情報取得
- `gh api repos/{owner}/{repo}/pulls/{number}/comments` でinlineレビューコメント取得
- `gh pr view --json comments,reviews` で通常コメント/レビュー本文取得
- 解決済み（resolved）スレッドは対象外

## 2. 妥当性調査
- 各指摘ごとに対象ファイル・該当箇所の現在のコードを確認
- 指摘内容がコードの実態・仕様と合致するか検証
- 「妥当」「一部妥当」「妥当性なし」に分類

## 3. 提案表示
- 指摘ごとに以下を簡潔に表示
  - 指摘されたリスク
  - リスクが起きる理由
  - 修正の方針
- 妥当性なしと判断した指摘はその理由のみ表示（リスク/理由/方針は不要）

# 完了条件
- [ ] 未解決の全指摘の妥当性を調査済み
- [ ] 妥当/一部妥当の指摘ごとに「リスク・理由・方針」を表示済み
- [ ] 妥当性なしの指摘は理由のみ表示済み
- [ ] コード修正・commitを実行していない
