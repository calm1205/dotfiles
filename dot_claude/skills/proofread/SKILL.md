# 役割
入力文章の推敲・新規作成文章の整形をproofread agentへ委譲

# 起動契機
- 明示的な推敲依頼（「推敲して」「文章を整えて」「読みやすくして」等）
- 文章作成タスク（Issue本文・PR説明・ドキュメント・議事録・コミットメッセージ等）の最終整形

# 入力
- `$ARGUMENTS` に対象文章本体・ファイルパス・該当箇所の指示が渡される
- 文章作成タスクでは、生成した下書きをそのまま渡す
- 引数が空ならユーザーへ対象を確認

# 手順

## 1. 対象の特定
- ファイルパスが渡された場合はReadで内容取得
- 文章が直接渡された場合はそのまま使用
- 範囲指示があれば該当部分のみ抽出

## 2. proofread agent呼び出し
- Agent toolで`subagent_type: proofread`を指定
- 対象文章をpromptに含める
- agent側の推敲方針（分量最小化・短文化・構造化・明確化）に委ねる

## 3. 結果の提示
- agentが返した推敲後の文章をそのまま出力
- 元のファイルへの書き戻しはユーザー指示時のみ実施

# ルール
- 原文の意味・出典・数値を改変しない
- 推敲後文章のみを出力（解説・差分説明を付けない）
- 書き戻し指示がない限りファイル編集は行わない
