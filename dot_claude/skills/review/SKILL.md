# 役割
未コミット変更またはPRに対する5観点並列コードレビューの実行

# ワークフロー

## 1. 差分取得

### モード分岐
- **PR番号/URLが引数にある場合** → `gh pr diff` + `gh pr view --json title,body,baseRefName,headRefName`
- **引数なし** → `git diff HEAD` + `git status`（未追跡ファイル含む）
- 変更がなければ「レビュー対象なし」と出力して終了

## 2. 差分サイズの判定
- テストファイル（`*_test.*`, `*.test.*`, `*.spec.*`, `test_*`, `tests/`, `__tests__/`）の差分行数を除外して判定
- テストファイル以外の修正が100行以下: エージェント1つで5観点を統合レビュー
- テストファイル以外の修正が100行超: 5エージェントで並列レビュー（ステップ3へ）

## 3. 並列レビュー
5エージェントをAgentツール（`subagent_type: "general-purpose"`）で**並列起動**。各エージェントには差分をプロンプトに含めて渡す。

### エージェント構成
| エージェント | 観点 |
|---|---|
| security-reviewer | OWASP Top 10:2025に基づくセキュリティ脆弱性の検出 |
| architecture-reviewer | CLAUDE.md・プロジェクト規約への準拠、責務分離、既存パターンとの整合性 |
| logic-reviewer | バグ・エッジケース未処理・エラーハンドリング漏れ・N+1クエリ等の検出 |
| test-reviewer | テスト網羅性の検証。テスト未変更の場合はテスト追加の必要性を指摘 |
| tradeoff-analyst | 変更が「何を得て何を犠牲にしたか」を**指摘ではなく問い**として出力 |

### 各エージェント共通の指示
- 差分のみをレビュー対象（必要に応じてGlob/Grep/Readで周辺コードを確認）
- 推測ではなくコードの事実に基づき、確信度の高い指摘に絞る
- 自分の担当観点のみ指摘（他観点と重複しない）
- 指摘ゼロなら「指摘なし」と返す

## 4. 結果集約・出力
- 全エージェント完了後、重複排除・重要度順に並べ替えて出力

```markdown
# レビュー結果

## Critical（修正必須）
- [セキュリティ/A01]: 説明 [file:line]

## Warning（推奨修正）
- [設計]: 説明 [file:line]

## トレードオフ
- **問い**: この変更で△△が犠牲になっていますが、意図通りですか？ [file:line]
  **得たもの**: 〇〇
  **犠牲**: △△
```

# ルール
- 指摘がないカテゴリは省略。全観点で指摘なしなら「問題なし」
- 各指摘に観点タグ（[セキュリティ], [設計], [ロジック], [テスト]）とファイルパス:行番号を付記
- セキュリティ指摘にはOWASPカテゴリ番号（A01〜A10）を付記
- PRモード時はヘッダーにPR番号・タイトルを表示
- 日本語で出力
