`incremental-implementation` skill を `test-driven-development` skill と併せて呼び出す。

## モード

- **`/build`** — *次の*保留タスクを実装し、停止する(慎重に、1スライスずつ)。
- **`/build auto`** — 必要ならプランを生成し、単一の承認を得たうえで、タスク間で止まらず*全*タスクを実装する。

`$ARGUMENTS` がモードを選ぶ。
`auto`(正式名)または `all` を自律モードとして扱い、それ以外(または空)はデフォルトの単一タスクモード。
注意: 自律モードは*タスク単位では*速くない——同じテスト駆動ループを回す——タスク*間*の人間の介在を除くだけ。

## デフォルト: 1タスク

プランから次の保留タスクを選ぶ。
そして:

1. タスクの受け入れ基準を読む
2. 関連コンテキストを読み込む(既存コード、パターン、型)
3. 期待する挙動に対する失敗するテストを書く(RED)
4. テストを通す最小限のコードを実装する(GREEN)
5. 全テストスイートを実行しリグレッションを確認
6. ビルドを実行しコンパイルを検証
7. 説明的なメッセージでコミット
8. タスクを完了にし、停止する

## 自律: プラン全体(`/build auto`)

スペックが存在し、プラン+ビルドを1回の実行にまとめたいときに使う。
タスク間の手動介在を除くが、検証は除か**ない**。
各タスクは依然として通過テストと自身のコミットを得る。

1. **スペックを必須とする。** 既知のパスのスペックのみを探す: リポジトリルートの `SPEC.md`、`docs/SPEC.md`、または `spec/` 配下のファイル。README や任意のドキュメントは**該当しない**。無ければ停止し、まず `/spec` を実行するようユーザーへ伝える——要件を捏造しない。
2. **クリーンなベースラインを確立する。** `git status --porcelain` を実行。想定される計画成果物(`SPEC.md`, `docs/SPEC.md`, `spec/*`, `tasks/plan.md`, `tasks/todo.md`)以外の未コミット変更があれば停止し、コミット・stash・対応方針をユーザーへ確認する。自律的なタスク単位コミットが無関係なローカル作業を吸収してはならない。さもないとクリーンロールバック保証が壊れる。
3. **必要ならプランを作る。** `tasks/plan.md` が無ければ `planning-and-task-breakdown` skill を呼び出して生成する。
4. **単一チェックポイント。** プラン全体を提示し、明確な肯定(例: 「承認」「go」「yes」)を待つ。曖昧な返答(「良さそう」「まあいいかな」)は承認と**みなさない**。これが唯一の人間のゲート——承認後は自律実行する。`tasks/plan.md` を生成した場合は、最初のタスクのコミットに混ざらないよう、単一の準備コミットとして今コミットする。
5. **依存順に全タスクを実行する。** 各タスクの宣言された依存を使う。明示的でなければプランの列挙順で実行する。各タスクでは上記デフォルトループ全体を回す(RED → GREEN → リグレッション → ビルド → コミット → 完了マーク)。そのタスクが触れたファイルとタスク状態更新のみをstageする——決して `git add -A` を無闇に使わない——タスクごとに1コミットし、どの時点もクリーンなロールバック地点にする。
6. **停止してユーザーへ確認する**(押し通さない)条件:
   - テストを通せない、または明白な修正なくビルドが壊れる → `debugging-and-error-recovery` skill に従う
   - スペックが曖昧、またはスペックが扱わない判断をタスクが要する
   - タスクが高リスク・不可逆——認証/権限変更、破壊的データマイグレーション、決済、削除、デプロイ、秘匿情報に触れるもの、**または `git revert` で戻せないもの** → `doubt-driven-development` skill に従い、続行前に明示的な承認を得る

   ユーザーがブロッカーを解消したら、`/build auto` を再呼び出しする——次の保留タスクから再開する。
7. **最後に要約する:** 完了タスク、追加テスト、作成コミット、スキップ/フラグ/ユーザー保留にした事項。

いずれかのステップが失敗したら、`debugging-and-error-recovery` skill に従う。
