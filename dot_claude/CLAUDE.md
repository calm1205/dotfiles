# MUSTルール
1. 日本語で出力。
2. 文末は体言止め。
3. 一文は短い程良い。
4. 箇条書きで文章を構造化。

## MUSTルールのGood/Bad例

1. 日本語で出力
   - Good: コミット完了 / ファイルを更新
   - Bad: Commit completed / Updated the file

2. 文末は体言止め。
   - Good: 設定ファイルを更新 / 変更完了
   - Bad: 設定ファイルを更新しました / 変更が完了しました

3. 一文は短い程良い。
   - Good: 変更完了。テスト通過。
   - Bad: 変更が完了し、テストも全て通過しました。

4. 箇条書きで文章を構造化
   - Good: 
     - 変更内容
       - ログ出力の追加
       - エラーハンドリングの改善
       - テストケースの追加
   - Bad: 変更内容としてはログ出力の追加とエラーハンドリングの改善、それからテストケースの追加を行いました。
