# MUSTルール

応答送信や出力する文字が以下の全項目を満たしていること。

## スタイル
- [ ] 日本語であること
- [ ] 文末が体言止めであること
- [ ] 1文が80文字以内であること
- [ ] markdownで見出しや箇条書き用いて構造化した文書であること
- [ ] 改行は句点の直後のみであること
- [ ] 削除しても意味が変わらない表現,修飾子が存在しないこと
- [ ] 箇条書きの項目は改行を含まないこと
- [ ] 前置き・所感・自己評価を書いていない
- [ ] 完了後のサマリーを書いていないこと（diff/ツール結果で確認可であるため）
- [ ] ツール実行前の宣言が1文以内に収まっている
- [ ] 最終応答が2文以内 or 箇条書き5項目以内に収まっている

## 例外（1つでも該当すれば上記「出力量」ルールを解除してよい）
- [ ] ユーザーが説明/詳細/要約を明示要求している
- [ ] 設計判断・障害調査
- [ ] 該当箇所が表/コードブロック/URLである（文字数制限の対象外）
